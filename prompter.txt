A rota PUT /api/v1/agents/:agentId já permite a
  atualização de um agente, e podemos modificá-la para incluir a atualização 
  do routerAgentId.

  Como funciona:

   1. Rota da API: A rota PUT /api/v1/agents/:agentId em
      src/api/agent.routes.js é a porta de entrada para a atualização de
      agentes.

   2. Controlador: O agentController.updateAgent em
      src/controllers/agent.controller.js recebe a requisição e os dados a
      serem atualizados.

   3. Lógica de Atualização: O controlador chama o prisma.agent.update para 
      efetivamente atualizar os dados do agente no banco de dados.

  O que precisa ser alterado:

  Para permitir a atualização do routerAgentId, precisamos fazer uma pequena
  alteração no agent.controller.js.

ler o agent.schema.js para ver se o routerAgentId já está no schema de  
  validação. Caso contrário, terá que adicioná-lo.
